    # blogs 表

    blogs 表的 userid 字段其实是外键，它会关联 users 表的 id 字段，且在设置外键时，要设置【删除时】和 【更新时】设为 CASCADE(级联，也就是当说删除 users 表中某个用户时候，其 blogs 也会相应被删除, 并且在往 blogs 表插入数据时，不能插入 userid 不存在的数据) 注意：后面我们使用 sequelize 建立外键关联时，它不是 CASCADE 而是 RESTRICT（即严格的，也就是说想要删除用户，得先把该用户的所有博客先删除！？） 注意：content 字段的类型设置为 TEXT

    # 多表查询

    -- blogs.* 表示要查出blogs表的所有字段
SELECT blogs.*, users.username, users.nickname FROM blogs INNER JOIN users on users.id = blogs.userid
