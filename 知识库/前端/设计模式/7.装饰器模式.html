    # 包装器

    装饰器模式是将一个对象嵌入另一个对象之中，实际上相当于这个对象被另一个对象包装起来，形成一条包装链。请求随着这条链条依次传递到所有的对象，每个对象有处理这个请求的机会。

    // 这是美式咖啡，啥都没加，只有水和咖啡，很苦很难喝
class Coffee {
  make(water) {
    // 黑咖啡只有水和咖啡
    return `${water}+咖啡`;
  }
  cost() {
    // 需要10元
    return 10;
  }
}

// 这是加奶的咖啡
class MilkCoffee {
  constructor(parent) {
    this.parent = parent;
  }
  make(water) {
    // this.parent.make(water)是原来的逻辑，原来的逻辑不变，只是增强了
    return `${this.parent.make(water)}+牛奶`;
  }
  cost() {
    return this.parent.cost() + 1;
  }
}
// 这是加糖的咖啡
class SugerCoffee {
  constructor(parent) {
    this.parent = parent;
  }
  make(water) {
    return `${this.parent.make(water)}+糖`;
  }
  cost() {
    return this.parent.cost() + 2;
  }
}
// 先拿到美式咖啡
let coffee = new Coffee();

let milkCoffee = new MilkCoffee(coffee);
let ret1 = milkCoffee.make("水");
console.log(ret1); // 输出：'水+咖啡+牛奶'

let milksugerCoffee = new SugerCoffee(milkCoffee);
let ret2 = milksugerCoffee.make("水");
console.log(ret2); // 输出： '水+咖啡+牛奶+糖'

let sugerCoffee = new SugerCoffee(coffee);
let ret3 = sugerCoffee.make("水");
console.log(ret3); // 输出：'水+咖啡+糖'
console.log(sugerCoffee.cost()); // 输出：12

    # AOP

    • 在软件业，AOP 为 Aspect Oriented Programming 的缩写，意为：面向切面编程
    • 可以通过预编译方式和运行期动态代理实现在不修改源代码的情况下给程序动态统一添加功能的一种技术
    • AOP 本质就是在函数执行之前或之后添加一些额外的逻辑，而不需要修改函数的功能

    // 买盐之前的逻辑
Function.prototype.before = function (beforeFn) {
  let _this = this;
  return function () {
    beforeFn.apply(this, arguments);
    _this.apply(this, arguments);
  };
};
Function.prototype.after = function (afterFn) {
  let _this = this;
  return function () {
    _this.apply(this, arguments);
    afterFn.apply(this, arguments);
  };
};
// 比如自己是男人，那么结婚之前，直接买
function buy(money, goods) {
  console.log(`花${money}买${goods}`);
  return "hello";
}

// 结婚之后，你想买盐，得先想媳妇申请钱
buy = buy.before(function () {
  console.log(`向媳妇申请1块钱.`);
});

// 买完盐之后，再做剩下的后续逻辑
buy = buy.after(function () {
  console.log(`把剩下的2毛钱还给媳妇.`);
});
buy(0.8, "盐");

    # 星巴克咖啡订单项目

    • 星巴克咖啡订单项目（咖啡馆）：
        1. 咖啡种类/单品咖啡：Espresso(意大利浓咖啡)、ShortBlack、LongBlack(美式咖啡)、Decaf(无因咖啡)
        1. 调料：Milk、Soy(豆浆)、Chocolate
        1. 要求在扩展新的咖啡种类时，具有良好的扩展性、改动方便、维护方便
        1. 使用 OO 的来计算不同种类咖啡的费用: 客户可以点单品咖啡，也可以单品咖啡+调料组合。

    # 星巴克咖啡订单项目---方案一

    最容易想到的方式就是所有的组合情况列举出来，但是这样的组合的方式很多，容易导致类爆炸

    # 分析方案一存在的问题

      1. Drink 是一个抽象类，表示饮料
      1. des 就是对咖啡的描述, 比如咖啡的名字
      1. cost() 方法就是计算费用，它在 Drink 类中会被做成一个抽象方法.
      1. Decaf 就是单品咖啡，它继承 Drink, 并实现 cost
      1. Espress && Milk 就是单品咖啡+调料，问题就是出现在这里，这个组合很多
      1. 存在的问题：这样设计，会有很多类，当我们增加一个单品咖啡，或者一个新的调料，类的数量就会倍增，就会出现类爆炸

    # 星巴克咖啡订单项目---方案二

    • 此方案稍好点
    • 前面分析到方案一因为咖啡单品+调料组合会造成类的倍增，因此可以做改进，将调料内置到 Drink 类，这样就不会造成类数量过多。从而提高项目的维护性(如图)
    • 说明: milk,soy,chocolate 可以设计为 Boolean,表示是否要添加相应的调料.

    # 分析方案二存在的问题

      1. 方案二可以控制类的数量，不至于造成很多的类
      1. 在增加或者删除调料种类时，代码的维护量很大
      1. 考虑到用户可以添加多份调料时，可以将 hasMilk 返回一个对应 int 类型
      1. 考虑使用 装饰者模式

    # 装饰者模式定义

      1. 装饰者模式：动态的将新功能附加到对象上。在对象功能扩展方面，它比继承更有弹性，装饰者模式也体现了开闭原则(ocp)
      1. 这里提到的动态的将新功能附加到对象和 ocp 原则，在后面的应用实例上会以代码的形式体现，请同学们注意体会。

    # 装饰者模式(Decorator)原理

      1. 装饰者模式就像打包一个快递
      • 主体：比如：陶瓷、衣服 (Component), 充当了被装饰者
      • 包装：比如：报纸填充、塑料泡沫、纸板、木板(Decorator)，充当了装饰者
      1. Component
      • 主体：比如类似前面的 Drink
      1. ConcreteComponent 和 Decorator
      • ConcreteComponent：具体的主体，比如前面的各个单品咖啡
      • Decorator: 装饰者，比如各调料.
      1. 在如图的 Component 与 ConcreteComponent 之间，如果 ConcreteComponent 类很多, 还可以设计一个缓冲层，将共有的部分提取出来，抽象层一个类。

    # 装饰者模式解决星巴克咖啡订单

    • 说明:
        1. Drink 类就是前面说的抽象类，就是 Component
        1. ShortBlack 就单品咖啡
        1. Decorator 是一个装饰类，含有一个被装饰的对象(Drink obj)
        1. Decorator 的 cost 方法进行一个费用的叠加计算，会递归的计算价格

    # 装饰者模式下的订单, 比如点了：2 份巧克力 + 一份牛奶的 LongBlack

    • 说明：
        1. Milk 包含了 LongBlack
        1. 一份 Chocolate 包含了(Milk+LongBlack)
        1. 一份 Chocolate 包含了(Chocolate+Milk+LongBlack)
        1. 这样不管是什么形式的单品咖啡+调料组合，通过递归方式可以方便的组合和维护
    export {};

abstract class Drink {
  public des!: string; // 描述
  private price!: number; // 价格

  public getDes(): string {
    return this.des;
  }

  public setDes(des: string) {
    this.des = des;
  }

  public getPrice(): number {
    return this.price;
  }

  public setPrice(price: number) {
    this.price = price;
  }

  // 计算费用的抽象方法
  // 由子类来实现
  public abstract cost(): number;
}

class Coffee extends Drink {
  public cost(): number {
    return super.getPrice();
  }
}

class Espresso extends Coffee {
  constructor() {
    super();
    this.setDes("意大利咖啡");
    this.setPrice(6.0);
  }
}

class LongBlack extends Coffee {
  constructor() {
    super();
    super.setDes("LongBlack");
    super.setPrice(5.0);
  }
}

class ShortBlack extends Coffee {
  constructor() {
    super();
    this.setDes("ShortBlack");
    this.setPrice(4.0);
  }
}

class Decorator extends Drink {
  private obj!: Drink;

  constructor(obj: Drink) {
    super();
    this.obj = obj;
  }

  public cost(): number {
    // super.getPrice() 是获取到自己的价格
    return super.getPrice() + this.obj.cost();
  }

  public getDes(): string {
    // this.obj.getDes() 是获取被装饰者的信息
    return super.getDes() + " " + super.getPrice() + " && " + this.obj.getDes();
  }
}

// 调味品类 Chocolate
class Chocolate extends Decorator {
  constructor(obj: Drink) {
    super(obj);
    this.setDes("巧克力");
    this.setPrice(3.0); // 调味品的价格
  }
}

// 调味品类 Milk
class Milk extends Decorator {
  constructor(obj: Drink) {
    super(obj);
    this.setDes("牛奶");
    this.setPrice(2.0); // 调味品的价格
  }
}

// 调味品类 Soy
class Soy extends Decorator {
  constructor(obj: Drink) {
    super(obj);
    this.setDes("豆浆");
    this.setPrice(1.5); // 调味品的价格
  }
}

// 1. 先点一份 LongBlack
let order: Drink = new LongBlack();
console.log("费用 =", order.cost());
console.log("描述 =", order.getDes());

// 2. 加入一份牛奶
order = new Milk(order);

console.log("order 加入一份牛奶 之后的费用 =", order.cost());
console.log("order 加入一份牛奶 之后的描述 =", order.getDes());

// 3. 加入一份巧克力
order = new Chocolate(order);
console.log("order 加入一份牛奶 加入一份巧克力 之后的费用 =", order.cost());
console.log("order 加入一份牛奶 加入一份巧克力 之后的描述 =", order.getDes());

// 4. 再加入一份巧克力
order = new Chocolate(order);
console.log(
  "order 加入一份牛奶 加入一份巧克力 加入一份巧克力 之后的费用 =",
  order.cost()
);
console.log(
  "order 加入一份牛奶 加入一份巧克力 加入一份巧克力 之后的描述 =",
  order.getDes()
